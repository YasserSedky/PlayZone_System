## 📋 المشاكل التي تم حلها

### 1. ✅ حساب المصروفات في تقرير الورديات

**المشكلة:**

- كان يتم حساب المصروفات بشكل خاطئ في تقرير الورديات
- عند وجود عدة فواتير لنفس الوردية، كانت المصروفات تتكرر بعدد الفواتير
- مثال: إذا كانت الوردية لديها 3 فواتير و 2 مصروف بقيمة 10 جنيه لكل منهما، كان المجموع يظهر 60 جنيه بدلاً من 20 جنيه

**الحل:**

- تم فصل حساب المصروفات عن الفواتير باستخدام subquery منفصلة
- تم تطبيق الإصلاح في 3 استعلامات:
  1. استعلام الورديات الرئيسي (`get_shift_analysis`)
  2. استعلام إحصائيات الورديات
  3. استعلام أداء الكاشيرز

**الملفات المعدلة:**

- `models/report_model.py` - السطور 663-735

**مثال الكود:**

```python
# قبل الإصلاح (خطأ)
COALESCE(SUM(e.amount), 0) as total_expenses
FROM shifts s
LEFT JOIN expenses e ON s.id = e.shift_id

# بعد الإصلاح (صحيح)
COALESCE((
    SELECT SUM(e.amount)
    FROM expenses e
    WHERE e.shift_id = s.id
), 0) as total_expenses
```

### 2. ✅ عرض تفاصيل المنتجات في الفاتورة

**المشكلة:**

- كانت تفاصيل المنتجات المضافة للفاتورة لا تظهر في نافذة تفاصيل الفاتورة
- كان يظهر فقط إجمالي سعر المنتجات دون التفاصيل
- السبب: الدالة `get_invoice_products` كانت تبحث فقط في جدول `invoice_items` ولا تبحث في `session_products`

**الحل:**

- تم تحديث دالة `get_invoice_products` للبحث في جميع الجداول المحتملة:
  1. `invoice_items` (المنتجات المحفوظة مباشرة مع الفاتورة)
  2. `session_products` (المنتجات المضافة أثناء الجلسة)
- يتم أولاً جلب `session_id` من الفاتورة
- ثم البحث في `invoice_items`، وإذا لم توجد منتجات، يتم البحث في `session_products`

**الملفات المعدلة:**

- `models/invoice_model.py` - السطور 479-523
- `models/payment_model.py` - السطور 424-468

**مثال الكود:**

```python
# الحل الجديد
# 1. جلب session_id
invoice_info = self.db.execute_query(
    """SELECT session_id FROM invoices WHERE id = ?""",
    (invoice_id,)
)

# 2. البحث في invoice_items
result = self.db.execute_query(
    """SELECT ii.*, p.name as product_name, p.category
       FROM invoice_items ii
       LEFT JOIN products p ON ii.product_id = p.id
       WHERE ii.invoice_id = ?""",
    (invoice_id,)
)

# 3. إذا لم توجد، ابحث في session_products
if not result and session_id:
    result = self.db.execute_query(
        """SELECT sp.*, p.name as product_name, p.category
           FROM session_products sp
           LEFT JOIN products p ON sp.product_id = p.id
           WHERE sp.session_id = ?""",
        (session_id,)
    )
```
